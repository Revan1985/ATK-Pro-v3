12. TROUBLESHOOTING E FAQ

────────────────────────────────────────────────────────────────────────────────

❓ DOMANDE FREQUENTI

🔵 P: Il programma si blocca al caricamento manifest?

R: Solitamente è timeout di rete. Verifica:

1. Connessione Internet: apri browser e accedi a https://www.google.com

2. URL corretto: copia direttamente da browser

3. Firewall: ATK-Pro deve essere consentito (dovrebbe chiedere al primo avvio)

4. Se in rete aziendale con proxy: configura proxy nelle impostazioni di rete del sistema operativo

Se problema persiste, prova con URL diverso per testare connessione IIIF

🔵 P: "File non trovato (404)" - cosa significa?

R: L'URL del manifest non è raggiungibile. Cause:

  • URL scaduto (repository non aggiornato)
  • Repository offline
  • Parametri URL errati (ID manifest sbagliato)

Soluzione: Verifica URL in browser prima di incollare in ATK-Pro

🔵 P: Download molto lento o si interrompe?

R: Prova:

1. Verifica router/WiFi (collega via cavo Ethernet se possibile)

2. Riavvia il router (spesso risolve problemi di connessione)

3. Prova con URL IIIF diverso per testare se il server è lento

4. Controlla velocità Internet con speedtest.net (min 2 Mbps consigliati)

5. Se solo alcuni server sono lenti, il problema non è di ATK-Pro

🔵 P: Ricostruzione immagini lenta - come accelerare?

R: La velocità dipende principalmente da:

  • Risoluzione IIIF del server (80-120 KB per tile è standard)
  • CPU disponibile (processo intensivo, 4+ core consigliati)
  • RAM disponibile (16 GB ideale, >4 GB necessario)
  • Velocità Internet (2+ Mbps consigliati)

Se molto lento: elabora record con meno pagine per volta

🔵 P: Output PDF è corrotto o non apre?

R: Prova:

1. Cancella PDF parziale della cartella output

2. Ricarica il record IIIF e rigenera PDF (Menu → File input → Apri input)

3. Verifica che almeno 5 GB di spazio libero siano disponibili

4. Se continua, prova visualizzare le immagini JPEG/PNG singole (non dipendono da PDF)

🔵 P: Come utilizzo i file TIFF per archivio?

R: TIFF è formato lossless, perfetto per conservazione:

1. Durante download, seleziona TIFF

2. Immagini generate in TIFF non-compresso

3. Backup su disco esterno (consuma molto spazio)

4. TIFF mantiene qualità indefinitamente

🔵 P: Posso elaborare più record contemporaneamente (in parallelo)?

R: No, ATK-Pro elabora un record alla volta (single-thread), non in parallelo. PERÒ puoi elaborare molti record in sequenza con una sola operazione:

1. Prepara un file input.txt con TUTTI i record (D o R, uno dopo l'altro)

2. Clicca Menu → File input → Apri input e seleziona il tuo file

3. Clicca Menu → Output → Elabora (una sola volta)

4. ATK-Pro elabora automaticamente tutti i record in sequenza, senza interruzioni

Esempio: Se input contiene 5 record, ATK-Pro processa automaticamente:

  • Record 1 (download + ricostruzione)
  • Record 2 (download + ricostruzione)
  • ... fino a Record 5

Tutti i record vengono salvati nelle cartelle che hai scelto al PASSO 2.

NON devi ricominciare l'elaborazione per ogni record.

🔵 P: File si salvano dove esattamente?

R: Percorso default:

  • Windows: C:\Users\[TuoUtente]\Documents\ATK-Pro\output\
  • Linux/macOS: ~/Documents/ATK-Pro/output/

Puoi cambiare cartella per ogni elaborazione (Menu → Output → Seleziona cartella output).

Cartella è creata automaticamente al primo run se non esiste.

🔵 P: Posso spostare file output dopo elaborazione?

R: Si, file sono indipendenti.

Dopo completamento:

1. Sposta cartella [recordID]/ dove desideri

2. PDF è auto-contenuto, non ha dipendenze

3. Se vuoi rielaborare lo stesso record, tieni manifest.json

🔵 P: Crashes improvvisi - cosa fare?

R: Se il programma crasha:

1. Controlla Log:
  • Windows: Documents\ATK-Pro\output\logs\*
  • Linux/macOS: ~/Documents/ATK-Pro/output/logs/*

2. Copia messaggio errore (CTRL+C sul messaggio)

3. Riavvia programma (spesso risolve)

4. Se crash ripetuto, prova su record IIIF diverso (potrebbe essere problema del manifest, non di ATK-Pro)

⚠️ ERRORI COMUNI E SOLUZIONI

Errore Causa Soluzione
"Network timeout" Server IIIF lento/offline Riprova con URL diverso
"Invalid IIIF format" URL non è Manifest IIIF Verifica URL in browser
"Disk space full" Spazio disco insufficiente Libera spazio su unità
"Permission denied" Cartella protetta/Read-only Windows: "Run as admin" / Linux-macOS: sudo
"URL not found (404)" Manifest IIIF scaduto Controlla ID nel Portale
"Window not responding" Elaborazione lenta/pesante Aspetta o clicca Annulla
