4. CARICAMENTO RECORD IIIF - FORMATI DI INPUT

────────────────────────────────────────────────────────────────────────────────

📥 STRUTTURA DELL'INPUT

ATK-Pro accetta file di input con un formato specifico basato su COPPIE: Per OGNI record da elaborare, fornisci:

1. RIGA 1: Modalità + Nome (facoltativo)

  • D = Documento singolo (una singola immagine/pagina)
  • R = Registro (collezione completa di pagine/documenti)

2. RIGA 2: URL del Manifest IIIF

Esempio di file input.txt corretto:

# Commenti iniziano con #

RECORD 1 (coppia di righe):
D - Lettera Importante
https://biblioteca.example.org/iiif/doc123/manifest.json

RECORD 2 (coppia di righe):
R - Registro Amministrativo 1920-1925
https://archives.example.org/iiif/reg456/manifest.json

RECORD 3 (coppia di righe):
D - Fotografia Singola
https://collections.example.org/iiif/foto789/manifest.json

📝 REGOLE FORMATO INPUT:

  • Righe vuote: ignorate (puoi usarle per leggibilità)
  • Righe con #: ignorate come commenti
  • Modalità D o R: sono obbligatorie (maiuscole)
  • Nome file: facoltativo dopo modalità (es. "D - Nome" o solo "D")
  • URL: deve iniziare con http:// o https://
  • Coppie: modalità + URL devono stare su righe consecutive

⚙️ COME CARICARE L'INPUT - 3 OPZIONI DAL MENU:

OPZIONE 1: Visualizza Esempio Input ──────────────────────────────────────

1. Clicca Menu → "File input" → "Esempio input"

2. Si apre una finestra che mostra il file di esempio localizzato

3. Puoi copiare il contenuto come template per il tuo file input

4. File di esempio: assets/[lingua]/testuali/input_link_base_.txt

OPZIONE 2: Apri File Input Pronto ──────────────────────────────────────

1. Clicca Menu → "File input" → "Apri input"

2. Si apre un dialog di selezione file

3. Scegli il tuo file input.txt (deve rispettare il formato coppie)

4. ATK-Pro carica automaticamente e legge tutti i record

5. Visualizza numero di record caricati e pronti per l'elaborazione

OPZIONE 3: Modifica Input Già Caricato ──────────────────────────────────────────

1. Dopo aver caricato un file input (OPZIONE 2)

2. Clicca Menu → "File input" → "Modifica input"

3. Si apre un editor di testo con il file attualmente caricato

4. Modifica le coppie (aggiungi, rimuovi, cambia URL)

5. Clicca "Salva" per aggiornare i record in elaborazione

6. Il programma reinterpreta il file aggiornato

📌 DIFFERENZA TRA D E R:

  • D (Documento): Gestisce una singola immagine/pagina
  • → Output: file singolo per il documento
  • R (Registro): Gestisce collezioni multi-pagina
  • → Output: cartella con pagine organizzate + PDF (se richiesto dall'utente)

✅ VALIDAZIONE RECORD

Dopo il caricamento, il programma verifica:

  • ✓ URL accessibile e raggiungibile
  • ✓ Formato IIIF valido (questa versione+)
  • ✓ Numero immagini e canvas disponibili
  • ✓ Metadata (titolo, autore, data)

❌ ERRORI COMUNI

  • "URL non valido": Controlla copia/incolla URL
  • "Manifest non trovato (404)": URL è scaduto o sbagliato
  • "Formato IIIF non supportato": Il server non usa IIIF standard
  • "Timeout connessione": Connessione Internet lenta/assente

💾 SALVATAGGIO RECORD

Dopo il caricamento del manifest, il programma salva automaticamente i metadati IIIF. DOVE: Nella cartella output che scegli (vedi Sezione 5, PASSO 2) STRUTTURA CARTELLE DOPO CARICAMENTO (prima dell'elaborazione):

[cartella_output]/
 ├─ manifest_[ID]_[nome].json         ← Manifest IIIF originale (livello principale)
 └─ [nome]/                            ← Sottocartella SOLO per metadati
    ├─ [nome]_genealogico.json        ← Metadati genealogici estratti
    └─ [nome]_tecnico.json            ← Metadati tecnici (canvas ID, numero pagine)

DOPO L'ELABORAZIONE (quando clicchi Menu → Output → Elabora): Nel livello principale vengono aggiunti i file elaborati:

[cartella_output]/
 ├─ manifest_[ID]_[nome].json         ← Manifest IIIF originale
 ├─ [nome]_canvas_1.jpg/.png/.tif     ← Immagini ricostruite dai tile IIIF (una per pagina)
 ├─ [nome]_canvas_2.jpg/.png/.tif       (formato dipende da quello selezionato)
 ├─ [nome]_canvas_N.jpg/.png/.tif       (N = numero totale di pagine/canvas)
 ├─ [nome].json                        ← Sidecar JSON per ogni immagine (metadati EXIF)
 └─ [nome].pdf                         ← PDF completo (vedi spiegazione sotto)

I metadati rimangono nella sottocartella [nome]/ come salvati al caricamento.

NOTA IMPORTANTE: Organizzazione file

  • Manifest IIIF: livello principale della cartella output
  • Immagini (canvas), PDF, Sidecar JSON: livello principale della cartella output
  • SOLO metadati tecnici e genealogici: salvati in sottocartella [nome]/
  • Le cartelle temporanee (tiles_canvas_X/) vengono eliminate automaticamente al termine
  • Questa struttura consente di gestire più record nella stessa cartella senza confusione

Esempio: Se elabori "Registro1", troverai:

  • manifest_[ID1]_Registro1.json (livello principale)
  • Registro1_canvas_1.jpg, Registro1_canvas_2.jpg, ... (livello principale)
  • Registro1.pdf (livello principale)
  • Registro1/ (sottocartella SOLO con metadati genealogico e tecnico)

5. ELABORAZIONE E DOWNLOAD IMMAGINI

────────────────────────────────────────────────────────────────────────────────

🎯 FLUSSO DI ELABORAZIONE

Il workflow di elaborazione prevede 3 passi:

PASSO 1: SELEZIONA FORMATI IMMAGINE

─────────────────────────────────────

1. Carica un file input (File input → Apri input): la finestra formati si apre automaticamente

2. In alternativa: Clicca Menu → "Impostazioni" → "Seleziona formati immagine"

3. Si apre una finestra di dialogo con 3 opzioni:

☐ PNG → Formato senza perdita (lossless), migliore qualità, file più grandi

☐ JPG → Formato compresso (lossy), buon compromesso qualità/dimensione

☐ TIFF → Formato archiviale, massima qualità, file molto grandi

Documento PDF

☐ PDF → Genera un PDF completo con tutte le pagine del registro o documento, in aggiunta ai formati immagine selezionati (o come unico output se nessun formato immagine è selezionato)

4. Seleziona almeno un formato immagine o PDF (puoi sceglierne più di uno)

5. Clicca "Conferma"

✅ I formati scelti vengono memorizzati e pre-selezionati automaticamente alla prossima apertura del programma.

📝 CONSIGLI:

  • Per preview/consultazione: JPG (veloce da aprire, minore spazio)
  • Per archivio conservativo: PNG (lossless, compromesso di spazio) o TIFF (massima qualità)
  • Per entrambi: seleziona sia PNG che TIFF
  • Per PDF: seleziona PDF insieme a uno o più formati immagine, oppure seleziona solo PDF per generare esclusivamente il documento PDF senza file immagine separati

PASSO 2: SELEZIONA CARTELLE OUTPUT

──────────────────────────────────────

1. La finestra cartelle si apre automaticamente dopo la scelta dei formati (quando carichi un file input)

2. In alternativa: Clicca Menu → "Impostazioni" → "Seleziona cartelle output"

3. Scegli la MODALITÀ di organizzazione delle cartelle:

⬤ Cartella unica per tutti → un'unica cartella per tutti i record (doc e registri)

⬤ Cartelle separate doc/reg → una cartella per tutti i documenti, una per tutti i registri

⬤ Cartella per ogni record → una cartella distinta per ciascun record nel file input

4. Dopo la scelta della modalità, seleziona le cartelle richieste (1, 2 o N finestre di dialog)

5. Al termine mostra il riepilogo delle cartelle scelte

✅ Modalità e cartelle vengono memorizzate. Alla prossima apertura di un file input, vengono ripristinate automaticamente se compatibili con il numero di record.

⚠️ Se il nuovo file input ha un numero diverso di Documenti/Registri rispetto alle cartelle salvate, viene mostrato un avviso e la selezione viene azzerata per richiedere nuove cartelle.

📌 RIEPILOGO MODALITÀ:

  • Cartella unica: Semplice, tutto in un posto. Adatto per elaborazioni veloci o test
  • Separate doc/reg: Organizzazione per tipo. Default: output/doc/ e output/reg/
  • Per ogni record: Massima flessibilità. Puoi usare drive diversi, NAS, cartelle personalizzate

PASSO 3: AVVIA ELABORAZIONE

─────────────────────────────

1. Clicca Menu → "Output" → "Elabora"

2. Verifica che:

✓ Almeno 1 formato immagine selezionato

✓ Almeno 1 record caricato

Se mancano questi requisiti, appare messaggio di errore:

❌ "Nessun formato selezionato" → Vai a PASSO 1

❌ "Nessun record caricato" → Vai a Caricamento Input (Sezione 4)

📺 FINESTRA DI ELABORAZIONE

Non appena clicchi "Elabora", si apre una finestra modale con:

📊 ELEMENTI VISIBILI:

1️⃣ TITOLO FINESTRA: "Elaborazione in corso"

  • Rimane in primo piano finché l'elaborazione non termina
  • Non puoi interagire con il resto della GUI

2️⃣ ETICHETTA PROGRESSO: "Elaborazione record: 2/5 — Nome File"

  • Mostra: record attualmente elaborato (2) su totale (5)
  • Mostra anche il nome del file/registro in elaborazione
  • Si aggiorna in tempo reale man mano che processa i record
  • Es: "Elaborazione record: 1/3 — Registro Matrimoni 1850"

3️⃣ BARRA DI PROGRESSO: Avanzamento visuale

  • Formato numerico: "2/5" (numero attuale / totale)
  • Barra colore che avanza da sinistra a destra
  • Raggiunge il 100% (massima larghezza) quando completo
  • NON mostra percentuale, solo numero

4️⃣ PULSANTE [ANNULLA]

  • Cliccabile durante elaborazione
  • Chiude la finestra di progresso
  • ⚠️ L'elaborazione continua in background (non interrompe il download)
  • I file scaricati rimangono nella cartella output selezionata

🔄 DURANTE L'ELABORAZIONE

Il programma in background:

├─ Scarica immagini dai server IIIF
├─ Legge manifest per canvas ID
├─ Ricostruisce tile in immagini complete
├─ Converte nei formati selezionati (PNG, JPG, TIFF)
└─ Salva nelle cartelle scelte

La GUI mostra progress in tempo reale:

  • Ogni 2-3 secondi aggiorna numero record
  • Nessun blocco: interfaccia rimane reattiva
  • Puoi annullare in qualsiasi momento

⚠️ COMPORTAMENTO SE CLICCHI ANNULLA:

1. Elaborazione si ferma subito

2. File parziali vengono rimossi

3. Finestra si chiude

4. Puoi ricominciare da Menu → Output → Elabora

📋 PDF — GENERAZIONE AUTOMATICA DURANTE L'ELABORAZIONE

Se hai selezionato PDF nella maschera di scelta formati (PASSO 1), il PDF viene generato automaticamente al termine del download di ogni registro o documento, senza nessuna finestra di conferma aggiuntiva.

📌 COMPORTAMENTO:

  • ✅ PDF selezionato nella maschera formati: il PDF viene generato automaticamente al termine del download; non viene chiesta nessuna conferma aggiuntiva
  • ❌ PDF non selezionato: solo le immagini nei formati scelti, nessun PDF

Caratteristiche del PDF generato:

  • Nome file: [nome].pdf, senza suffissi aggiuntivi (es. Registro Matrimoni 1920.pdf)
  • Salvato nella cartella output scelta, allo stesso livello delle immagini
  • Pagine in ordine numerico crescente (canvas 1 → canvas 2 → … → canvas N)
  • Se un canvas non è scaricabile, viene inserita una pagina placeholder localizzata con link diretto al canvas originale su Antenati
  • Nessun file temporaneo rimane nella cartella output al termine della generazione

⚠️ COSA NON FARE DURANTE ELABORAZIONE:

  • ❌ Non chiudere il programma (perde file parziali)
  • ❌ Non staccare disco esterno (errore salvataggio)
  • ❌ Non spegnere PC/laptop
  • ❌ Non staccare internet (arresta download)

✅ COSA PUOI FARE:

  • ✓ Aspettare che finisca (è la procedura normale)
  • ✓ Cliccare Annulla se urgente
  • ✓ Usare il PC per altre cose normalmente (minimizza solo la finestra)

🎉 FINE ELABORAZIONE

Quando tutti i record sono stati elaborati:

1. La finestra di progresso si chiude automaticamente

2. Appare un messaggio: "Operazione completata"

3. Clicca OK per chiudere il messaggio

4. La finestra di progresso scompare completamente

Dopo:

  • Visualizza file output nelle cartelle che hai selezionato al PASSO 2
  • Ogni documento/registro ha propria sottocartella all'interno della cartella da te scelta
  • Immagini sono pronte per uso (visualizzazione, stampa, archiviazione, PDF)

✅ OUTPUT GENERATO:

Per ogni record elaborato troverai:

  • Immagini nei formati selezionati (PNG, JPG, TIFF)
  • manifest_[ID]_[nome].json → Manifest IIIF originale con metadati strutturati
  • [nome]_genealogico.json → Metadati genealogici (date, luoghi, nomi, comuni, province)
  • [nome]_tecnico.json → Metadati tecnici (numero canvas, lista canvas ID, diritti, file generati)
  • [nome].json → Sidecar JSON per ogni immagine (metadati EXIF embedded)

📝 NOTA SUI NOMI FILE:

  • Il [nome] base viene preso direttamente dal file input.txt (riga utente)
  • Se scrivi nel file input "Registro Matrimoni 1920", i file saranno:
    • Registro Matrimoni 1920_canvas_1.jpg
    • Registro Matrimoni 1920_genealogico.json
    • Registro Matrimoni 1920_tecnico.json
  • I suffissi JSON rimangono in EN (_genealogico, _tecnico) per compatibilità
  • Puoi usare nomi in qualunque lingua nel file input (IT, EN, misto)

6. PDF PER REGISTRI E DOCUMENTI

────────────────────────────────────────────────────────────────────────────────

COME ATTIVARE LA GENERAZIONE PDF

La generazione del PDF si attiva selezionando la casella PDF nella maschera di scelta formati (Menu → Impostazioni → "Seleziona formati immagine").

Puoi combinare PDF con uno o più formati immagine (es. PDF + PNG), oppure selezionare solo PDF per ottenere esclusivamente il documento senza file immagine separati.

📄 CARATTERISTICHE DEL PDF GENERATO

  • Nome file: il PDF è denominato semplicemente [nome].pdf, senza suffissi aggiuntivi — ad esempio Registro Matrimoni 1920.pdf
  • Posizione: salvato nella cartella output scelta, allo stesso livello delle immagini: [cartella_scelta]\[nome].pdf
  • Ordine pagine: le pagine sono sempre in ordine numerico crescente (canvas 1 → canvas 2 → … → canvas N), indipendentemente dall'ordine in cui i tile vengono scaricati
  • Pagine placeholder: se un canvas non è disponibile (errore di rete o risorsa temporaneamente non raggiungibile), al suo posto viene inserita automaticamente una pagina placeholder localizzata nella lingua selezionata. La pagina placeholder contiene:
    • Un testo di avviso localizzato (es. "Immagine non disponibile")
    • Il link diretto al canvas originale su Antenati (https://antenati.cultura.gov.it/ark:/…), così da poter consultare la fonte in modo semplice
    In questo modo viene sempre generato il PDF, con tutte le informazioni necessarie a completarlo in caso di immagini non scaricate.
  • Pulizia automatica: i file temporanei utilizzati per la generazione del PDF vengono eliminati automaticamente al termine del processo. Nella cartella output rimangono solo le immagini nei formati scelti, il PDF finale, i manifest e i metadati

🚫 Se PDF non è selezionato:

  • Solo le immagini vengono scaricate nei formati selezionati
  • Nessun PDF generato

9. CARTELLE E FILE OUTPUT

────────────────────────────────────────────────────────────────────────────────

📂 STRUTTURA OUTPUT

Quando elabori un record IIIF, ATK-Pro salva i file nella cartella che hai scelto nel PASSO 2 (Seleziona cartella output).

STRUTTURA ORGANIZZATA PER RECORD:

[cartella_output]/
├─ manifest_[ID]_[nome].json         ← Manifest IIIF originale
├─ [nome]_canvas_1.jpg/.png/.tif     ← Immagini ricostruite (una per pagina)
├─ [nome]_canvas_2.jpg/.png/.tif
├─ [nome]_canvas_N.jpg/.png/.tif     ← (N = numero totale pagine/canvas)
├─ [nome].json                       ← Sidecar JSON per ogni immagine (EXIF)
├─ [nome].pdf                        ← PDF completo (solo se richiesto)
└─ [nome]/                           ← Sottocartella SOLO per metadati
   ├─ [nome]_genealogico.json        ← Metadati genealogici
   └─ [nome]_tecnico.json            ← Metadati tecnici

MULTI-RECORD NELLA STESSA CARTELLA:

Se elabori 3 record nella stessa cartella, la struttura diventa:

[cartella_output]/
├─ manifest_[ID1]_Registro1.json
├─ Registro1_canvas_1.jpg
├─ Registro1_canvas_2.jpg
├─ Registro1.json (sidecar)
├─ Registro1.pdf
├─ Registro1/
│  ├─ Registro1_genealogico.json
│  └─ Registro1_tecnico.json
├─ manifest_[ID2]_Documento2.json
├─ Documento2_canvas_1.jpg
├─ Documento2.json (sidecar)
├─ Documento2.pdf
├─ Documento2/
│  ├─ Documento2_genealogico.json
│  └─ Documento2_tecnico.json
├─ manifest_[ID3]_Registro3.json
├─ Registro3_canvas_1.jpg
├─ Registro3_canvas_2.jpg
├─ Registro3.json (sidecar)
├─ Registro3.pdf
└─ Registro3/
    ├─ Registro3_genealogico.json
    └─ Registro3_tecnico.json

📌 NOTE SULLA STRUTTURA:

  • Manifest IIIF, Immagini, PDF, Sidecar JSON: nel livello principale
  • SOLO metadati tecnici e genealogici: in sottocartelle [nome]/ (uno per record)
  • Le cartelle temporanee (tiles_canvas_X/) e i file temporanei utilizzati per la generazione del PDF vengono eliminati automaticamente al termine dell'elaborazione: nella cartella output non rimane nessun file di lavoro intermedio
  • Questa organizzazione consente di gestire molteplici record nella stessa cartella senza confusione tra file

📋 CONVENZIONE DI DENOMINAZIONE:

  • Manifest: manifest_[ID]_[nome].json
  • Immagini: [nome]_canvas_1, [nome]_canvas_2, ... [nome]_canvas_N
  • JSON metadati: [nome]_genealogico.json, [nome]_tecnico.json
  • JSON immagini: [nome].json (per ogni canvas)
  • PDF: [nome].pdf
